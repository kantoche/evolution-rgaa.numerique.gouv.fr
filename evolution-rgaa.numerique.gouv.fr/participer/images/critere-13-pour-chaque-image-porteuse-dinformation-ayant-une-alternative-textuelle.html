Critère 1.3 [A] Pour chaque image porteuse d'information ayant une alternative textuelle, cette alternative est-elle pertinente (hors cas particuliers) ?

  • Proposition DINSIC | 08/03/2019 - 16:13

    Modification du test 1.3.1

    Raison de la modification du test 1.3.1

    Prise en compte de l’ensemble des techniques permettant d’associer une alternative textuelle à une balise <img>.

    Ancien test 1.3.1

    Test 1.3.1 : Chaque image (balise img) porteuse d'information, ayant un attribut alt, vérifie-t-elle ces conditions (hors cas particuliers) ?

    • Le contenu de l'attribut alt est pertinent.
    • S'il est présent, le contenu de l'attribut title est identique au contenu de l'attribut alt.
    • S'il est présent, le contenu de la propriété aria-label est identique au contenu de l'attribut alt.
    • S'il est présent, le contenu du passage de texte lié via la propriété aria-labelledby est identique au contenu de l'attribut alt.

     

    Nouveau test 1.3.1

    Test 1.3.1 : Chaque image (balise <img> ou balise possédant l’attribut WAI-ARIA role="img") porteuse d'information, ayant une alternative textuelle, vérifie-t-elle ces conditions (hors cas particuliers) ?

    • S’il est présent, le contenu de l'attribut alt est pertinent.
    • S'il est présent, le contenu de l'attribut title est pertinent.
    • S'il est présent, le contenu de l’attribut WAI-ARIA aria-label est pertinent.
    • S'il est présent, le passage de texte associé via l’attribut WAI-ARIA aria-labelledby est pertinent.

    Signalez un abus

    Il y a de nombreux tests pour les critères liés aux images qui ont été modifiés pour indiquer que les contenus des attributs title, aria-label, aria-labelledby doivent être pertinents. Auparavant, on indiquait "identique".

    Sur quoi se base la pertinence alors ? Les tests ne sont plus du tout écrits de façon explicite et vont rendre leur bonne application très complexe.

    Si la pertinence est toujours basée sur le fait que ça doive être identique à l'attribut "alt", dans ce cas, il vaudrait mieux laisser "identique" plutôt que "pertinent".

    Si la pertinence se base sur d'autres critères d'appréciation, dans ce cas, ceux-ci devraient être précisés, a minima, dans une entrée du glossaire.

    DINSIC 13/03/2019 - 10:42

    Bonjour, l'utilisation de l'attribut alt n'étant plus la seule technique autorisée, il est nécessaire de vérifier que l'ensemble des techniques pouvant être utilisée fournissent une alternative pertinente. Lorsque plusieurs techniques sont utilisées conjointement la valeur restituée sera de cette manière forcement pertinente. De plus, il n'y a dans WCAG aucune obligation qu'elles soient toutes identiques si elles sont pertinentes. Il appartiendra alors à l'auditeur de juger de la pertinence de chacune de valeurs utilisées.
