    Signalez un abus

    Dans la mesure où l'attribut longdesc est désormais non-conforme (cf. https://www.w3.org/TR/html52/obsolete.html#element-attrdef-img-longdesc), il pourrait sembler souhaitable de retirer la condition "La description détaillée via l'adresse référencée dans l'attribut longdesc est pertinente.".

    De la même façon que la balise acronym, ou l'attribut summary sur les tableaux ont été supprimés.

    Par ailleurs, il suffit qu'une de ces conditions soit renseignée, il ne devrait pas être réclamé que toutes ces conditions soient remplies.

    Je propose donc la réécriture du test comme suit :

    Test 1.7.1 : Chaque image (balise ) porteuse d'information, ayant une description détaillée, vérifie-t-elle une de ces conditions ?
    - La description détaillée dans la page et signalée par l’alternative textuelle est pertinente.
    - La description détaillée via un lien ou bouton adjacent est pertinente.
    - Le passage de texte associé via l’attribut WAI-ARIA aria-describedby est pertinent.

    DINSIC 22/03/2019 - 11:44

    Bonjour, bien qu'étant obsolète en html 5.2, l'attribut reste utilisable dans les versions précédentes de html. Par ailleurs, WCAG n'interdisant pas l'usage d'élément obsolètes (dans la mesure où ils ne causent pas de problèmes d'accessibilité bien entendu voir cas particulier 8.2) cette solution reste totalement utilisable tout comme l'usage de l'attribut summary (voir nouvelle definition de glossaire resume).

    Concernant l'obligation de respecter l'ensemble des conditions cela est nécessaire car plusieurs techniques peuvent être utilisées simultanément. Néanmoins les conditions devrait être modifiée selon la tournure suivante :

    • Si elle est présente, la description détaillée via l'adresse référencée dans l'attribut longdesc est pertinente.
    • Si elle est présente, la description détaillée dans la page et signalée par l’alternative textuelle est pertinente.
    • Si elle est présente, la description détaillée via un lien ou bouton adjacent est pertinente.
    • Si elle est présente, le passage de texte associé via l’attribut WAI-ARIA aria-describedby est pertinent.
